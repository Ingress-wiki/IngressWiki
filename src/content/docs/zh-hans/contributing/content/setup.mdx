Ingress.Wiki 使用[Starlight](https://starlight.astro.build)和[Astro](https://astro.build)构建。为了在向[我们的GitHub仓库](https://github.com/Ingress-wiki/IngressWiki)贡献之前预览你的改动，你需要设定一个开发环境。

:::note
对于修正拼写错误这类的小改动，你可以使用GitHub网页界面进行编辑。每个页面底部都有一个「编辑此页」链接。
:::

如果你打算经常贡献或已经安装好大多数工具，可以在自己的电脑上搭建环境；但如果你不熟悉编程，也可以使用基于云的开发环境，例如[GitHub Codespaces](https://github.com/features/codespaces)。

<Tabs>
  <TabItem label="本地计算机" icon="laptop">
    <Steps>
      1. 安装[Node.js](https://nodejs.org/en/download)和[pnpm](https://pnpm.io/installation#using-a-standalone-script)。

         我们使用pnpm作为包管理器。即使npm也能安装依赖，我们仍然推荐使用pnpm来避免潜在的依赖版本冲突。

      2. 克隆我们的GitHub仓库。如果你不熟悉Git，我们推荐使用[GitHub Desktop](https://github.com/apps/desktop)，它有相对易用的界面。

         仓库地址: `https://github.com/Ingress-wiki/IngressWiki`

      3. 在仓库目录中运行`pnpm install`来安装依赖。

      4. 运行`pnpm dev`来启动开发服务器。你会看到类似下面的输出：

         ```
         astro  v5.2.5 ready in 1473 ms

         ┃ Local    http://localhost:4321/
         ┃ Network  use --host to expose
         ```

         在浏览器中打开输出中「Local」后面的地址来查看站点。
    </Steps>
  </TabItem>

  <TabItem label="GitHub Codespaces" icon="github">
    <Steps>
      1. 在浏览器中打开这个GitHub[仓库](https://github.com/Ingress-wiki/IngressWiki)。

      2. 点击「Code」按钮，选择「Codespaces」选项卡并创建一个新的Codespace。

      3. 等待它加载。如果它提示你安装Astro扩展，请点击「Install」。

      4. 在终端中运行`pnpm install`安装依赖。

      5. 运行`pnpm dev`启动开发服务器。

      6. 你会看到弹窗提示类似以下内容：

         > Your application running on port 4321 is available.

         点击「Open in Browser」来查看站点。
    </Steps>
  </TabItem>
</Tabs>

现在，你可以开始编辑`src/content/docs`目录中的 Markdown 文件来进行贡献。保存文件后，站点会自动更新。之后你可以在GitHub上创建一个Pull Request来提交你的改动。
